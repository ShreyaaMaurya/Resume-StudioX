⚡ QUICK FIX - Package Version Issue

═══════════════════════════════════════════════════════════════════════════════

WHAT HAPPENED:

  npm error code ETARGET
  npm error notarget No matching version found for jsonwebtoken@^9.1.2.

The package versions in package.json don't exist in npm registry.

═══════════════════════════════════════════════════════════════════════════════

WHAT I DID:

✅ Updated package.json with compatible versions
✅ jsonwebtoken: ^9.0.2 (instead of ^9.1.2)
✅ express: ^4.18.2 (instead of ^5.2.1)
✅ mongoose: ^8.0.0 (instead of ^9.6.2)

═══════════════════════════════════════════════════════════════════════════════

WHAT YOU NEED TO DO:

Step 1: STOP THE SERVER
────────────────────────
  Press: Ctrl + C


Step 2: DELETE OLD FILES
──────────────────────────

Option A (Windows CMD):
  rmdir /s node_modules
  del package-lock.json

Option B (PowerShell):
  Remove-Item -Recurse node_modules
  Remove-Item package-lock.json

Option C (Manual):
  1. Delete the node_modules folder
  2. Delete the package-lock.json file


Step 3: CLEAR NPM CACHE
─────────────────────────
  npm cache clean --force


Step 4: INSTALL AGAIN (NOW IT WILL WORK!)
──────────────────────────────────────────
  npm install

  Wait 2-3 minutes until you see:
  "added 156 packages"


Step 5: START SERVER
──────────────────────
  npm run dev

  Should see:
  [nodemon] starting `node server.js`
  [DATABASE] MongoDB Connected Layer: localhost
  🚀 ResumeBuilder API Server
  ✨ Running on http://localhost:5000


Step 6: TEST IT!
────────────────
  1. Open http://localhost:5000/signUp.html
  2. Create test account
  3. Open http://localhost:5000/login.html
  4. Login
  5. Access resume builder ✓
