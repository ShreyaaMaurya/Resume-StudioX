================================================================================
                        KEY FEATURES IMPLEMENTED
================================================================================

SIGNUP:
  • Collect: First Name, Last Name, Email, Password
  • Validate: Password confirmation, minimum 8 characters
  • Process: POST /api/auth/register
  • Result: User stored in MongoDB with hashed password
  • Storage: JWT tokens stored in localStorage
  • Redirect: Auto-redirect to /Resume.html

LOGIN:
  • Collect: Email, Password
  • Process: POST /api/auth/login
  • Verify: Credentials checked against MongoDB
  • Result: User authenticated
  • Storage: JWT tokens stored in localStorage
  • Redirect: Auto-redirect to /Resume.html

PROTECTED ACCESS:
  • Check: accessToken in localStorage
  • Logic: IF token found → Allow access
          IF token NOT found → Redirect to login
  • Result: Only authenticated users access resume builder

================================================================================
                        DATABASE & SECURITY
================================================================================

MongoDB Collection: users

Fields Stored:
  • fullName (required)
  • email (required, unique)
  • password (hashed with bcryptjs, 10 salt rounds)
  • phoneNumber (optional)
  • profileImage (optional)
  • refreshTokens (array with 7-day TTL)
  • createdAt, updatedAt (timestamps)

Security Features:
  ✅ Passwords hashed (bcryptjs - 10 salt rounds)
  ✅ JWT tokens with expiration (30min access, 7day refresh)
  ✅ Email validation (RFC standard)
  ✅ CORS protection
  ✅ Protected API endpoints
  ✅ Token stored server-side for validation
  ✅ Automatic token expiration

================================================================================
                        API ENDPOINTS
================================================================================

PUBLIC ENDPOINTS:
  POST /api/auth/register       - Create new account
  POST /api/auth/login          - Login user
  POST /api/auth/refresh        - Get new access token
  POST /api/auth/forgot-password - Request password reset

PROTECTED ENDPOINTS (require Bearer token):
  GET /api/auth/me                   - Get current user
  PUT /api/auth/update-profile       - Update profile
  POST /api/auth/change-password     - Change password
  POST /api/auth/logout              - Logout user
  POST /api/auth/reset-password/:token - Complete password reset
