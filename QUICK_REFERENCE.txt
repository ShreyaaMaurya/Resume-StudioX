    -H "Content-Type: application/json" \
    -d '{

      "email": "test@example.com",

      "confirmPassword": "TestPass123"
    }'

LOGIN USER:
  curl -X POST http://localhost:5000/api/auth/login \
    -H "Content-Type: application/json" \
    -d '{
      "email": "test@example.com",
      "password": "TestPass123"
    }'

CREATE RESUME (use token from login):
  curl -X POST http://localhost:5000/api/resume \
    -H "Content-Type: application/json" \
    -H "Authorization: Bearer YOUR_TOKEN_HERE" \
    -d '{
      "fullName": "Test User",
      "targetRole": "Developer",
      "email": "test@example.com",
      "blocksData": []
    }'

🔐 AUTHENTICATION FLOW
═══════════════════════════════════════════════════════════════════════════════

1. Register
   POST /api/auth/register → Get accessToken + refreshToken

2. Login
   POST /api/auth/login → Get accessToken + refreshToken

3. Use Access Token
   GET /api/resume (with Authorization header)
   Authorization: Bearer <accessToken>

4. Refresh Token (when access token expires)
   POST /api/auth/refresh → Get new accessToken

5. Logout
   POST /api/auth/logout → Invalidate tokens

🚨 COMMON ISSUES
═══════════════════════════════════════════════════════════════════════════════

MongoDB Connection Error:
  Solution: Make sure mongod is running ($ mongod)

Port Already in Use:
  Solution: Change PORT in .env or kill process using port 5000

JWT Token Expired:
  Solution: Use refresh token endpoint to get new access token

CORS Error:
  Solution: Add your domain to CORS_ORIGIN in .env

Missing Required Fields:
  Solution: Check API_DOCUMENTATION.md for required fields
